# 🟦 SLIDE 5 — SOLUTION OVERVIEW

## 🏗️ SSPA Architecture

* 🧬 Survivor Identity Vault (offline-first encrypted identity)
* 📦 Evidence Hash Store (append-only, tamper-evident)
* 🌐 Distributed Relay Nodes (NGO / field hubs)
* ⛓️ Ledger Anchoring (EVM-compatible Merkle root settlement)

---

### 🎙️ Narasi:

> “SSPA separates identity storage, evidence capture, and blockchain anchoring into independent resilient layers.”

---

# 🟦 SLIDE 6 — HOW IT WORKS

## 🔄 Data Flow

1. Evidence captured on device
2. SHA-256 hash generated locally
3. Stored offline in immutable log
4. Sync via NGO relay (when available)
5. Anchored as Merkle Root on-chain

---

### 🎙️ Narasi:

> “Even if a device is destroyed, the cryptographic proof of what happened still survives.”
